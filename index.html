01왜 스킬을 만들게 됐나

같은 프롬프트를 5번 이상 치고 있다면, 그건 스킬이 되어야 합니다.

저는 AI 교육·컨설팅 프리랜서로 일하면서 매일 비슷한 패턴이 반복됩니다. 아침에 여러 클라이언트 워크스페이스를 훑어보고, 미팅 장소 간 이동 일정을 넣고, 읽어야 할 트윗·유튜브를 정리하고, 하루 끝에 작업을 요약합니다. 각 작업마다 30초~15분씩 잡아먹혔습니다.

Claude Code의 Skill 기능을 발견한 뒤로는 반복되는 건 전부 SKILL.md 한 장으로 옮겼습니다. 지금은 9개가 매일 돌아갑니다.

이 페이지의 목적 — 각 스킬이 어떤 반복 문제를 없앴는지, 어떻게 호출하는지, 어떤 결과물이 나오는지를 9개 전부 나열합니다. 본인 업무에서 비슷한 반복이 있다면 구조를 그대로 가져가실 수 있도록.

06스킬은 어떻게 만드나 — SKILL.md 한 장

Claude Code의 스킬은 놀랍게도 Markdown 파일 하나입니다. frontmatter로 트리거를 선언하고, 본문에 절차를 서술형으로 적으면 끝.

name: my-context-sync
description: 강수혁의 컨텍스트 싱크. Slack, Gmail, Google Calendar에서
             최근 정보를 수집하고 하나의 문서로 정리한다.
             "싱크", "sync", "정보 수집" 요청에 사용.
triggers:
  - "싱크"
  - "sync"
  - "정보 수집"
---

# My Context Sync
흩어진 정보를 한곳에 모아 정리하는 스킬.
...

설계 원칙 4가지

  • 사전 검증 먼저 — 환경변수 · API 키가 없으면 실행 중단, 설정 가이드만 출력 (실패 조기 탐지)

08이 강의 자체가 스킬로 만들어진 과정

AI·SW마에스트로 멘토특강 기획도 위 도구들로 만들어졌습니다.

  • 1단계 수집 — Playwright 기반 스크래퍼로 SWMaestro 멘토링 게시판 800건 자동 수집
  • 2단계 분석 — 어떤 주제가 반복되는지, 어떤 멘토 프로필이 인기인지 통계화
  • 3단계 기획 — 분석 결과 기반으로 "내가 가진 강점 × 빈 주제" 교집합 도출 → 5건 특강 기획
  • 4단계 발행 — HTML 작성 → Notion/게시판 임베드 안전 버전 별도 생성 → 자동 업로드
즉, 이 강의는 "자동화한 도구로 자동화를 가르친다"의 예시 그 자체입니다. 강의 끝나고 여러분이 돌아가서 본인 업무에 같은 패턴(수집 → 분석 → 기획 → 발행)을 얹으실 수 있도록 구조를 설계했습니다.

09자주 받는 질문 (Q&A 유도)

"Skill을 만들 만큼 반복되는 일이 별로 없는데요?"

처음엔 그렇게 느낍니다. 저도 첫 스킬(session-wrap) 이후 3주 동안 2개뿐이었습니다. 반복은 하고 나서야 보입니다. 먼저 Claude Code를 일상 업무에 꾸준히 쓰기 시작하면, 2~3주 뒤 "어 또 이거 시키네" 하는 순간이 옵니다. 그 때 SKILL.md로 옮기면 됩니다.

"API 키가 필요한 스킬이 많은데 다 발급해야 하나요?"

필요한 것만 발급하시면 됩니다. route-planner는 NCP·ODsay, find-restaurant는 네이버 개발자센터 검색 API 따로. 대부분 무료 티어로 개인 사용은 충분합니다.

"MCP는 꼭 써야 하나요?"

아닙니다. 저는 Slack·Gmail·Calendar는 claude.ai의 Connectors로 연결했는데, 이게 MCP 서버를 감싸준 형태입니다. Connectors로 시작하시고, 연결이 없는 서비스만 직접 MCP 서버를 세우시면 됩니다.

"직접 질문하고 싶어요"

준비해오신 자동화 아이디어 2~3가지 기준으로 Q&A 합시다. "내가 X를 매일 하는데 이게 스킬이 될까요?" — 이 질문이 가장 빠른 학습 루트입니다.
